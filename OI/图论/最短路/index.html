# Floyd 算法

用于求多源汇最短路（任意两个结点之间的最短路）。

复杂度较高，适用于任何图，不管有向无向，边权正负，但是最短路必须存在，不能有负环。

# 实现

定义数组 f[k][i][j] 表示只允许经过结点 111 到 kkk （注意 xxx 与 yyy 不一定在其中）结点 iii 到 jjj 的最短路长度。

则 f[n][i][j] 就是结点 iii 到结点 jjj 的最短路长度。

f[0][i][j] 的值：

  1. 当 iii 与 jjj 间有直接相连的边的时候，为边权；
  2. 当 i=ji=ji=j 的时候为 000 ，因为到本身的距离为 000 ；
  3. 当 iii 与 jjj 没有直接相连的边时，为 +∞+\infty+∞ 。

f[k][i][j] = min(f[k-1][i][j], f[k-1][i][k] + f[k-1][k][j]) ，

其中 f[k-1][i][j] 为不经过 kkk 点的最短路，而 f[k-1][i][k] + f[k-1][k][j] 为经过了 kkk 点的最短路。

这样，需依次增加问题规模（ kkk 从 111 到 nnn ），判断任意两点在当前问题规模下的最短路。

如果放在一个给定第一维 kkk 二维数组中， f[i][k] 与 f[k][j] 在某一行和某一列。而 f[i][j] 则是该行和该列的交叉点上的元素。
f[k][i][j] 的涵义是第一维为 k−1k-1k−1 这一行和这一列的所有元素的最小值，包含 f[k-1][i][j] ，那么在原地进行更改也不会改变最小值的值，因为如果将该三维矩阵压缩为二维，则所求结果 f[i][j] 一开始即为原 f[k-1][i][j] 的值，最后依然会成为该行和该列的最小值。

故第一维对结果无影响，数组的第一维是可省略，于是可改成 f[i][j] = min(f[i][j], f[i][k] + f[k][j]) 。

# Dijkstra 算法

用于求解非负权图上单源最短路径。

# 实现

将结点分成两个集合：已确定最短路长度的点集 SSS 集合的和未确定最短路长度的点集 TTT 集合。开始所有点都属于 TTT 集合。

开始时 diss=0dis_s=0diss​=0 ，其他点的 disdisdis 均为 +∞+∞+∞。

然后重复这些操作：

  1. 从 TTT 集合中，选取一个最短路长度最小的结点，移到 SSS 集合中。
  2. 对那些刚刚被加入 SSS 集合的结点的所有出边执行松弛操作。直到 TTT 集合为空，算法结束。

即

  1. 初始化 dis[s] = 0 ，其余节点为正无穷（一般使用 memset(dis, 0x3f, sizeof(dis)) ）。标记所有节点为未访问。
  2. 找出所有未被访问的节点中， dis[x] 最小的节点 ，此时的 dis[x] 即为点到点的最短路长度。然后标记访问 。
  3. 遍历点 的所有出边 ，若 dis[x] + w < dis[v] ，则以 dis[x] + w 更新 dis[v] 。
  4. 重复步骤 ，直到所有点都被标记为已访问。

有多种方法来维护 111 操作中最短路长度最小的结点，不同实现导致时间复杂度上的差异。本文的优先队列优化中，如果同一个点的最短路被更新多次，先前更新时插入的元素不能被删除和修改，故时间复杂度为 O(mlog⁡m)O(m\log m)O(mlogm) ，一般在非负权稀疏图上使用。

# SPFA 算法

SPFA 算法是 Bellman-Ford 算法的队列优化，主要应用于有负边权时求单源最短路，可判断有没有负环产生。

如果没有负边权，推荐使用 Dijkstra 算法。

Bellman-Ford 算法不断尝试对图上每条边进行松弛，每进行一轮循环，就对图上所有边都尝试进行一次松弛操作，当一次循环中没有成功的松弛操作时，算法停止。在最短路存在的情况下，由于一次松弛操作会使最短路的边数至少 +1+1+1 ，而最短路的边数最多为 n−1n-1n−1 ，因此整个算法最多执行 n−1n-1n−1 轮松弛操作。

但很多时候并不需要那么多无用的松弛操作。显然只有上一次被松弛的结点所连的边才有可能引起下一次松弛操作。 SPFA 用队列维护「哪些结点可能会引起松弛操作」，只访问必要的边。故 SPFA 一般情况下时间复杂度约为 O(m)O(m)O(m) ，最坏情况下为 O(mn)O(mn)O(mn) 。

如果从 sss 点出发，抵达一个负环时，松弛操作会无休止地进行下去。 SPFA 只需记录最短路经过了多少条边，当经过了至少 nnn 条边时，说明 sss 点可以抵达一个负环。

# 实现

已知一个起点，以及点之间的距离，还需要定一个队列，在队列中存放后面可以用来进行松弛操作的点，数组 cntcntcnt 表示某个点进入队列的次数。

从开始节点遍历边，当遍历到的点不在队列中的时候，将点放入队列， cnt++ ，并且更新最短距离，当某一个节点进入队列次数大于 nnn 的时候，就可以判定图中存在负环。

# 分层图最短路

若图中有些边是特殊边，这些特殊边只能走 kkk 次，直接在图上跑无法判断特殊边用了多少次，且转移较麻烦。就有了分层图。

例题 洛谷 P4568 [JLOI2011] 飞行路线

题意概要：给定一个有 nnn 个点、 mmm 条无向边的图，每条边有一个边权，其中有 kkk 次机会将一条普通边转化为一条边权为 000 的特殊边，求 sss 到 ttt 的最短路。

# Floyd

记原图中 u,vu,vu,v 之间边的边权为 mpu,vmp_{u,v}mpu,v​ 。

又 Floyd 算法有一性质：在最外层循环到点 kkk 时（尚未开始第 kkk 次循环），最短路数组 fff 中， fu,vf_{u,v}fu,v​ 表示的是从 uuu 到 vvv 且仅经过编号在 [1,k)\left[1, k\right)[1,k) 区间中的点的最短路。
